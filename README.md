# 青云界 — 全 AI 驱动修真世界模拟器

v3 架构。摒弃传统游戏引擎，由 DeepSeek API 驱动叙事生成、NPC 决策和世界演化。纯前端，无后端服务。

## 功能

- **世界卡系统** — 基底 + 扩展模组自由组合，构建不同的修真世界
- **AI 叙事引擎** — 玩家文字输入 → AI 生成剧情叙事 + 系统变更
- **世界流（World Flow）** — 每 N 次交互自动触发一轮世界推进：NPC 自主行动、外交编排、势力演化、时间推进
- **NPC 系统** — 每个角色有独立状态、性格、关系网、记忆压缩，支持自由对话（传书）
- **修仙体系** — K/P 等级、灵石/丹药/灵草/符箓等多资源系统
- **战斗系统** — AI 驱动的回合制战斗，支持战前 buff 和成长
- **命石系统** — 结算世界时根据表现获得命石，解锁更多世界卡扩展
- **OPFS 持久化** — 浏览器文件系统存储，自动读写，无后端
- **API 日志** — 完整记录每次 AI 调用，方便调试

## 技术栈

| 层 | 技术 |
|---|---|
| 构建 | Vite 8 |
| 语言 | TypeScript + JavaScript |
| 移动端 | Capacitor 8 (Android) |
| AI | DeepSeek API |
| 存储 | OPFS (浏览器文件系统) |

## 快速开始

```bash
npm install
npm run dev
```

浏览器打开 `http://localhost:5173/ui/setup.html`，填入 DeepSeek API Key 后即可开始。

## 项目结构

```
data/        世界观数据层（配置驱动）
transform/   AI 转化层（prompt 拼装）
engine/      引擎层（世界观无关的纯逻辑）
ui/          前端展示层
├── setup.html     游戏准备（世界卡选择、API Key、存档管理）
├── game.html      主游戏页面
├── chat.html      NPC 对话（传书系统）
└── api-log.html   API 调用日志
```

## 架构

世界流主循环（每 N 次玩家交互触发）：
```
大事件检查 → 世界 AI 叙事 → 信息派发 → NPC 个人行动 → 外交编排 → 状态编排 → 认知同步 → CRUD 执行 → 时间推进
```
